# LN-ZK580D300T28DNJG 扩展需求与实现说明

> **标准通信协议**以 `通信协议.docx` 为准（ICD：0x0401/0x0403/0x0412 等）。  
> 本文档仅描述**协议未定义、本项目额外实现**的功能：风机 PWM 主从转发细节、EEPROM 掉电保存、板卡参数设置/清零（0x0404）、0x0403 清零标志扩展等。  
> 对接外部主控软件时，需确认对方是否支持本文“厂商扩展”消息。

---

## 1. 与标准协议的关系

| 功能 | 标准协议（`通信协议.docx`） | 本项目扩展 |
|------|---------------------------|------------|
| 风机开关 + 占空比 | **0x0403**，占空比 `0x00/0x10~0xA0` | 一致 |
| 清零最大/最小值 | **0x0403**，`clear_flag = 0x01` | 一致 |
| 板卡编号/出厂日期/工作时长上报 | **0x0412** 消息体字段 | 一致（数据来源改为 EEPROM） |
| 清零工作时长 | 无 | `0x0403` 扩展 `clear_flag` 或 **0x0404** |
| 设置板卡编号/出厂日期/工作时长 | 无 | **0x0404** |
| EEPROM 掉电保存 | 无 | AT24C08（48PIN 主机） |
| 48PIN ↔ 28PIN 内部帧 | 无（板间私有） | `55 AA C1 / 上传 21 字节` |

---

## 2. 系统架构

```
外部主控 (PB10/PB11, SLIP+CRC)
        │ 0x0403 风机控制 / 0x0404 板卡参数 / 0x0401→0x0412
        ▼
48PIN 主机 (GD32C10x)
  ├─ 风机电源使能、协议解析、EEPROM、工作时长累计
  └─ PA2/PA3 内部串口 ──────────────────► 28PIN 从机
                                              ├─ ADC 采集上传
                                              └─ TIMER0 PWM → 风机转速
```

| MCU | 工程目录 | 对外串口 | 内部串口 |
|-----|----------|----------|----------|
| 48PIN 主机 | `LN-ZK580D300T28DNJG-host` | USART2 PB10/PB11 | USART1 PA2/PA3 |
| 28PIN 从机 | `LN-ZK580D300T28DNJG-Slave` | — | USART1 PA2/PA3 |

## 3. 风机 PWM 与风速控制（标准 0x0403 + 内部转发）

### 3.1 外部协议 0x0403（与 `通信协议.docx` 表 3 一致）

SLIP 解包后的**应用层帧**（不含 CRC，长度 14 字节）：

| 偏移 | 字段 | 说明 |
|------|------|------|
| 0 | 消息类型 | `0xF0` 设置 |
| 1 | 设备编号 | `0x00` |
| 2~3 | 消息 ID | `0x0403`（大端） |
| 4~5 | 数据长度 | `0x0010`（16，含帧头+消息体+CRC 的算法以协议为准） |
| 6 | 整机风机开关 | `0x00` 关电，`0x01` 开电 |
| 7 | 占空比档位 | `0x00`=0%，`0x10`~`0xA0` 步进 10%，低 4 位须为 0 |
| 8 | 清零标志 | 见 [§5](#5-0403-清零标志扩展) |
| 9~12 | 预留 | 默认 0 |

处理流程（主机 `main.c` → `Power_HandleControl`）：

1. 校验帧长、开关、占空比、`clear_flag` 合法范围  
2. `Power_SetFan()`：主机侧风机电源使能  
3. `Power_SendSlaveControl()`：经内部串口下发从机  
4. 按 `clear_flag` 执行最大最小值/工作时长清零（扩展）  
5. 回复 **0x0410** ACK/NAK  

### 3.2 占空比 → PWM 比较值（28PIN）

从机 `Power_SetFanControl()`：

```text
pwm_value = (duty_code >> 4) * POWER_PWM_PERIOD / 10
F_PWM_SetValue(pwm_value)   // TIMER0 CH0，POWER_PWM_PERIOD = 999
```

- 实现文件：`LN-ZK580D300T28DNJG-Slave/HARDWARE/PWM.c`（`F_PWM_SetValue`）  
- 配置入口：`LN-ZK580D300T28DNJG-Slave/USER/power.c`（`Power_SetFanControl`）  

### 3.3 内部控制帧（48PIN → 28PIN）

| 字节 | 内容 |
|------|------|
| 0~1 | `0x55 0xAA` |
| 2 | `0xC1` 控制命令 |
| 3 | 风机开关（同 0x0403） |
| 4 | 占空比（同 0x0403） |
| 5 | 清零标志（主机下发，从机当前仅用于风机控制逻辑） |
| 6 | `0x00` 预留 |
| 7~8 | `0x66 0xBB` |

- 发送：`host/USER/power.c` → `Power_SendSlaveControl()`  
- 接收：`Slave/USER/power.c` → `Power_HandleHostCommandFrame()`  

### 3.4 内部上传帧（28PIN → 48PIN）

固定 **21 字节**：`55 AA` + 采集数据 + `66 BB`（F28VU、风速 ADC、I_F1、Y、F_PWM 比较值、S0/S1/S2、温度等）。  
详见 `Slave/USER/power.h` 中 `POWER_UPLOAD_FRAME_LEN`。

## 4. EEPROM 掉电保存（厂商扩展）

### 4.1 硬件

- 器件：**AT24C08**（I2C 地址 `0xA8`）  
- 接口：软件 I2C，**PB14 = SCL，PB15 = SDA**（与从机工程引脚定义相同，请在 48PIN 原理图确认）  
- 驱动：`host/SYSTEM/eeprom/iic.c`、`24cxx.c`  
- 业务：`host/USER/power_eeprom.c`  

### 4.2 存储布局（基地址 0x0000，共 16 字节）

| 偏移 | 长度 | 内容 |
|------|------|------|
| 0~1 | 2 | 魔数 `0xA5 0x5A` |
| 2 | 1 | 版本 `0x01` |
| 3 | 1 | CRC8（对偏移 2~15 求和） |
| 4~9 | 6 | 板卡编号（与 **0x0412** `buff[7..12]` 同格式，按协议位域/BCD 填写） |
| 10~12 | 3 | 出厂日期（与 **0x0412** `buff[13..15]` 同格式） |
| 13~15 | 3 | 累计工作时长（小时，大端 24bit，与 **0x0412** `buff[74..76]` 一致） |

**默认值**（首次上电或 CRC 无效时写入）：

| 项目 | 默认内容 |
|------|----------|
| 板卡编号 | `20 26 01 00 01 00`（示例，需按产线规则修改） |
| 出厂日期 | `20 26 05`（2026 年 05 月，位域编码） |
| 工作时长 | `00 00 00` |

### 4.3 运行时行为

| 时机 | 行为 |
|------|------|
| 上电 `SYS_Init()` | `Power_EepromInit()` → `Power_EepromLoad()` → `Power_SetWorkHour()` |
| 后台每秒 | `Power_StatusReportTick()`，满 1 小时则 `power_work_hour++` 并 `Power_EepromSave()` |
| 0x0404 设置/清零 | 更新缓存后 `Power_EepromSave()` |
| 0x0412 上报 | `Power_EepromGetBoardId()` / `GetMfgDate()`，工作时长用 `power_work_hour` |

## 5. 0x0403 清零标志扩展

标准协议仅定义 **`clear_flag = 0x01`** 清零电流/电压最大最小值。本项目扩展如下（`power_eeprom.h`）：

| `clear_flag` | 含义 |
|--------------|------|
| `0x00` | 不清零 |
| `0x01` | 清零最大/最小值（**标准**） |
| `0x02` | 清零累计工作时长并写 EEPROM |
| `0x03` | 同时执行 `0x01` + `0x02` |

> 若对接方仅支持标准协议，请只使用 `0x00` / `0x01`；清工作时长请改用 **0x0404**。

## 6. 0x0404 板卡参数设置/清零（厂商扩展）

- **消息 ID**：`0x0404`（`power.h` → `POWER_MSG_BOARD_CFG`）  
- **消息类型**：`0xF0` 设置  
- **帧体长度**：14 字节（与 0x0403 相同 SLIP 帧结构）  
- **应答**：**0x0410** ACK/NAK（`ack_msg_id = 0x0404`）

### 6.1 设置子命令（`frame[6]`）

| `frame[6]` | 名称 | `frame[7..]` 数据 |
|------------|------|-------------------|
| `0x01` | 设置累计工作时长 | 3 字节，大端，单位 h |
| `0x02` | 设置板卡编号 | 6 字节（与 0x0412 板卡编号字段一致） |
| `0x03` | 设置出厂日期 | 3 字节（与 0x0412 出厂日期字段一致） |

### 6.2 清零子命令（`frame[6]`）

| `frame[6]` | 名称 |
|------------|------|
| `0x10` | 清零工作时长 |
| `0x11` | 恢复默认板卡编号 |
| `0x12` | 恢复默认出厂日期 |
| `0x1F` | 全部恢复默认（含工作时长） |

清零类命令无需填充 `frame[7..]`。

### 6.3 示例（应用层帧十六进制，不含 SLIP/CRC）

**设置占空比 50% 并开风机（标准 0x0403）**

```text
F0 00 04 03 00 10  01  50  00  00 00 00 00
     │     │       │   │   └─ clear_flag
     │     │       │   └─ duty 50%
     │     │       └─ fan on
     └─ len=16（按协议填写）
```

**设置工作时长为 100h（扩展 0x0404）**

```text
F0 00 04 04 00 10  01  00 01 64  00 00 00
                   └─ sub=0x01   └─ 0x000164 = 100
```

**清零工作时长（扩展 0x0404）**

```text
F0 00 04 04 00 10  10  00 00 00 00 00 00
                   └─ sub=0x10
```

## 8. 调试与测试建议

1. **风机**：发 0x0403，`fan=0x01`，`duty=0x50`，用示波器看 28PIN PWM 引脚占空比是否约 50%。  
2. **内部通信**：主机打印 `power_slave_f_pwm`，应与从机 `F_PWM_Value` 一致。  
3. **EEPROM**：设置板卡编号后发 0x0401 查询，0x0412 中编号应一致；断电再上电仍一致。  
4. **工作时长**：可 0x0404 写入较小值，或长时间运行后核对 0x0412 小时字段与 EEPROM。  
5. **兼容性**：对接标准主控时，勿发 0x0404 及 `clear_flag>0x01`，避免 NAK 或未知行为。
